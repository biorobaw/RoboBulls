Welcome to the RoboBulls 2 Documentation.

Installation

The required libraries and tools to run the game can be installed by running documents/scripts/install_robobulls.sh found with the Git download. By default, it installs in $HOME, but check out the script to change it. if "robobulls2" is not the installation path, it will also be downloaded.

The Installation Script will download and compile the following.

  • All required libraries to compile and run the code and tools
  • The grSim Simulator, the program used to simulate a game.
  • The SSL Referee Box, the standard control platform for the game.
  • The RoboBulls2 Project, the project itself.

Configuration

The project is controlled by a number of configuration files. See the Config Module in Modules. The three most important are:

  • include/config/simulated.h–Simulated or Field play– The SIMULATED macrco selects to run the project on the field (ENG lab) or grSim simulator.
  • include/config/team.h–Team Selection–The TEAM macro selects TEAM_BLUE or TEAM_YELLOW.
  • include/config/communication.h–Network config–Sets the addresses and ports of the communication modules

Getting Started--Running The Project

Here we are going to compile and run the project on the simulator and perform a simple Behavior.

1) Open the RoboBulls.pro project in Qt Creator(4.5.0) and compile. There are some issues to notice here, before you actually build the project, make sure you updated your QT creator and QT to the latest version. The last test is based on QT creator 4.5.0 and QT 5.10.0, make sure you set the QT version in your QT creator by going into Tools->Options->Build & Run->Kits. After running the installation script, there should be no compilation issues (Tested on Ubuntu 14.04 only)
2) Open the grSim simulator (grSim/bin/grsim) from the script's install path on your computer or any other computer
3) Go to include/config/communication.h and change SIMULATOR_ADDRESS to the IP (in quotes) of the computer running grSim (SIMULATOR_ADDRESS_LOCAL for local).
4) Change REFBOX_LISTEN_ENABLED to 0–This will enable of the TestStrategy below.
5) Change VISION_ADDRESS under the #if SIMULATED to the "Vision multicast address" shown on the left-hand side of the simulator
6) Go to strategy/teststrategy.cpp and insert gameModel->findMyTeam(0)->assignBeh<GoToBeh>(); under TestStrategy::assignBeh
7) Making sure SIMULATED in include/config/simulated is 1, Run the program.

If these steps are completed correctly, Robot 0 on TEAM should move to the center of the field.

8) For running the yisirobot in real world, simply change the SIMULATED into 0. Also make sure you connected the transmitter to your computer, the two lights flashing means that they are not successfully configured, they will turn into always lighting if they get configured. Use the command line to add robot (e.g... Add Robot:call add_robot <id> <team=b|y>)
