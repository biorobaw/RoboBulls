Detailed Description

Configuration Constants.

These are constants in include/config which control the game.

Macro Definition Documentation

#define REFBOX_LISTEN_ENABLED   1

Referee Box–do we listen to commands? See StrategyController.

Chooses between listening to RefComm, and always choosing TestStrategy.

#define SIMULATED   1

The SIMULATED constant, either 1 or 0.

This configuration macro is used to compile the RoboBulls project using the grSim simulator or the real-world vision cameras. Since both are mostly the same, most changes take place in the /communication module, but you can also use this constant to better-define tolerances such as distance tolerances.

  • define 0 = Not simulated; real world
  • define 1 = Simulated; grSim
#define TEAM   TEAM_BLUE

The Program's Team (TEAM_BLUE or TEAM_YELLOW)

The player's team constant. This sets which team the program will transmit to. XBee Channel D is for Blue, E is for Yellow

Variable Documentation

int GOALIE_ID

The goalie robot's ID is given as an integer.

This ID can be changed with the goalie_id integer on the utility command line. It is 5 by default (see .cpp)

const int ROT_TOLERANCE

The max difference in angle for two angles in radians to be considered equal.

Test this with Measurements::angleDiff. This is also the default tolerance for the angle version of Measurements::isClose
